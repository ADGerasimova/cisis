## 🎯 План действий

### Этап 1: Models (20 минут)
```bash
# 1. Резервная копия
cp core/models.py core/models.py.backup

# 2. Копирование готовых файлов
cp -r cisis_refactoring/models core/

# 3. Переименование старого
mv core/models.py core/models.py.old

# 4. Проверка
python manage.py makemigrations --dry-run
# Должно вывести: "No changes detected"
```

### Этап 2: Views (1-2 часа)
```bash
# 1. Резервная копия
cp core/views.py core/views.py.backup

# 2. Копирование структуры
cp -r cisis_refactoring/views core/

# 3. Перенос кода (используя VIEWS_REFACTORING_MAP.md)
# - Откройте старый views.py
# - Откройте VIEWS_REFACTORING_MAP.md
# - Копируйте код согласно карте

# 4. Обновление urls.py
# См. пример в VIEWS_REFACTORING_MAP.md

# 5. Переименование старого
mv core/views.py core/views.py.old

# 6. Проверка
python manage.py runserver
```

### Этап 3: Admin (30-60 минут)
```bash
# 1. Резервная копия
cp core/admin.py core/admin.py.backup

# 2. Создание структуры
mkdir core/admin
touch core/admin/__init__.py
touch core/admin/base_admin.py
touch core/admin/user_admin.py
touch core/admin/sample_admin.py
touch core/admin/permissions_admin.py
touch core/admin/logs_admin.py

# 3. Перенос кода (используя ADMIN_REFACTORING_MAP.md)
# - Откройте старый admin.py
# - Откройте ADMIN_REFACTORING_MAP.md
# - Копируйте классы согласно карте

# 4. Переименование старого
mv core/admin.py core/admin.py.old

# 5. Проверка
python manage.py runserver
# Откройте /admin/ и проверьте все модели
```

## 💡 Полезные советы

1. **Начните с models** - это самое простое и не требует переноса кода
2. **Сделайте git commit** после каждого этапа
3. **Не удаляйте .old файлы** минимум 2 недели
4. **Тестируйте после каждого этапа** - не делайте всё сразу
5. **Используйте карты** - там точные номера строк, не надо искать

## ⚠️ Что НЕ нужно делать

❌ Копировать всё сразу без проверки
❌ Удалять старые файлы models.py, views.py, admin.py
❌ Пропускать этап тестирования
❌ Менять структуру БД (managed = False!)
❌ Создавать новые миграции (должно быть "No changes detected")

## ✅ Критерии успеха

Рефакторинг прошёл успешно если:
- ✅ `python manage.py makemigrations` выводит "No changes detected"
- ✅ Сервер запускается без ошибок
- ✅ Админка открывается и все модели видны
- ✅ Журнал образцов работает
- ✅ Все URL маршруты работают
- ✅ Права доступа работают корректно

## 🆘 Поддержка

Если что-то пошло не так:
1. Посмотрите раздел Troubleshooting в `REFACTORING_GUIDE.md`
2. Проверьте что все .old файлы на месте
3. Сравните импорты в ваших файлах с примерами в картах
4. Убедитесь что `managed = False` во всех моделях (кроме SampleFile)
